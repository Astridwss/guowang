# 业务线一：

## 1、数据处理

### 文档流：无大模型（纯 Python 脚本 extract_from_word.py 拆解结构；将 Word 拆成图文切片；extract_from_word.py（负责读取 Word、按标题层级切片并提取 Base64 图片）。

## 2、结构化提炼（提取问题与答案）

### 处理语音转写：Qwen/QwQ-32B-AWQ (大语言模型)；dx_sim.py 或未开启 RAG 的 opt_asr.py（内部包含调用 QwQ 大模型进行【问题+业务域+原因+操作步骤】的提取 Prompt）

### 处理 Word 图文切片：Qwen/Qwen2.5-VL-32B-Instruct-AWQ (多模态视觉大模型)；optdocx_api.py（负责将 extract_from_word.py 的结果拼装，送入 Qwen-VL 视觉大模型提取标准步骤）。

### 描述：将杂乱的内容统一提炼为【问题】+【业务域】+【原因】+【操作步骤】+【菜单路径】。

## 3、知识向量化（Embedding）

### 模型：Alibaba-NLP/gte-Qwen2-7B-instruct (专门的 Embedding 向量模型)；extract_features.py（负责调用 gte 模型，将海量提炼出的文本转换为高维向量并保存为 .pkl 特征文件）。

### 描述：将提炼出的文字“问题”转换为计算机能计算距离的高维数学向量。

## 4、数据降维与聚类

### 模型：无（使用传统机器学习算法 t-SNE/PCA 降维，HDBSCAN/KMeans 聚类）；对应脚本：cluster.py（负责读取向量文件，执行 t-SNE/PCA 和 HDBSCAN 算法，并生成 3D 聚类可视化 HTML 文件）。

### 描述：在 3D 空间内把相似的向量聚成一个簇（Cluster），发现高频业务问题。

## 5、存入《标准 FAQ 知识库》

### 模型：Qwen/QwQ-32B-AWQ。

### 描述：通过算法取中心点，或者让大模型根据簇内数据总结出一个最完美的【标准问题+标准答案】，正式入库；cluster.py（该脚本在聚类完成后，会将每个簇的统计信息及代表性样本输出为 Excel，作为最终的 FAQ 知识库底表）。

# 业务线二：

## 1、新话务接入（听声辨字）

### 语音流：pyannote/speaker-diarization-3.1 (音频切分) + SenseVoiceSmall (语音转字)；将录音变成文字剧本；speaker-diarization.py（或 pyannote-whisper/speaker-divorce.py，负责调用 pyannote 进行音频切分）以及 api.py（SenseVoice 的本地接口服务，负责语音转字）。

### 模型：pyannote/speaker-diarization-3.1 + SenseVoiceSmall。

### 描述：将今天的新电话转成带有角色和情绪标签（如 <|SAD|>）的文字剧本。

## 2、意图提炼与 RAG 检索（辅助与纠偏）

### 模型：

    提炼问题：Qwen/QwQ-32B-AWQ。
    匹配检索：Alibaba-NLP/gte-Qwen2-7B-instruct (算向量与余弦相似度)。
    生成最终话术：Qwen/QwQ-32B-AWQ (结合检索到的 FAQ 给出最终参考或评价)。

### 描述：大模型（QwQ）提炼出新问题，向量模型（gte）去知识库里“搜”出标准 FAQ 答案，最后 QwQ 结合标准答案生成最终结果。

### 对应脚本：核心逻辑在 opt_asr.py（开启 --rag 1 参数时）和 dx_sim.py（负责计算余弦相似度）。它们联动完成意图提取、在 FAQ 库中检索相似度，并生成最终对比结果。

## 3、客户满意度打分（情感分析）

### 模型：Qwen/QwQ-32B-AWQ。

### 描述：大模型结合第一步 SenseVoice 吐出的情绪标签和对话上下文，对通话进行全局 0-100 的打分，并总结出客户痛点。

### 对应脚本：raw_asr_sentiment_analysis.py（并发调用 QwQ 大模型，根据预设的加减分规则，输出包含“情感评分”、“情感结论”和“改进建议”的 sentiment_analysis.xlsx）。

## 4、隐私安全抹除（脱敏）

### 模型：Qwen/QwQ-32B-AWQ。

### 描述：大模型识别文本中的订单、姓名、账号等敏感词，替换为 [ACCOUNT_1] 等系统占位符，生成安全文本。

### 对应脚本：desensitization.py（调用 QwQ 大模型执行正则替换，将敏感信息转化为占位符，输出 desensitization.xlsx）。

## 5、归档备案

### 模型：无。

### 描述：系统工程操作，将脱敏打分后的最终数据，存入企业数据库保存留档。

4 个核心 API：

一个语音 API (Tongyi Tingwu Pro) 解决切分和转写（序号 1、2）。sdk调用 jar包提供

一个视觉 API (Qwen-VL-14B) 解决文档图片解析（序号 3）。

## 模型：Qwen/QwQ-32B-AWQ

## 输出：剥离口语化表达后的标准纯文本结构，包含：提炼出的核心“问题”、“所属业务域”、“原因”、“操作步骤”、“菜单路径”等

## 提示词：

    请详细分析以下客服对话文本，并精准提炼出用户咨询的**问题** 、 **所属业务域** 和对应的 **答案**。

## 输入：gte 模型生成的多个高维向量

## 输出：将高维向量压缩为可在 3D 空间中展示的低维坐标 (x, y, z)

## 提示词：

## 对应脚本：cluster.py

    该脚本会读取前面生成的 .pkl 特征文件，调用 sklearn.manifold.TSNE 或 sklearn.decomposition.PCA（当样本量较少时会自动切换为 PCA），将高维向量压缩为可在 3D 空间展示的 (x, y, z) 坐标。

## 输入：降维后的向量点（通常是提取出的海量“问题”特征）

## 输出：聚类标签（将相似的请求归为一个簇（Cluster ID），噪声数据被标记为 -1）

## 提示词：

## 对应脚本：cluster.py

    降维完成后，该脚本继续调用 hdbscan.HDBSCAN 或 sklearn.cluster.KMeans 算法对数据点进行聚类。它会自动识别并剔除噪声点（标记为 -1），并将各个簇（Cluster）的统计信息（如样本数、中心点坐标）输出到 Excel 中，最后利用 plotly.express 生成一个可交互的 3D 可视化 HTML 文件。
